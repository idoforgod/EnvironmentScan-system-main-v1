# Longform Journalism Transformation Skill

## 개요

환경 스캐닝 보고서(통합 일일 보고서, 주제별 심층 보고서)를 장편 저널리즘 형식의
한국어 기사로 변환한다. 원본 보고서의 모든 분석적 내용을 보존하면서(무손실 변환),
서사적 구조와 저널리즘적 문체를 적용하여 가독성과 설득력을 극대화한다.

---

## 핵심 원칙

### 1. 무손실 변환 (Lossless Transformation)

원본 보고서의 모든 핵심 정보가 최종 기사에 반드시 포함되어야 한다.

- 모든 신호의 핵심 사실(core fact)은 본문에 포함
- 모든 출처, 방법론, 신뢰도 데이터는 미주(endnote)로 이동
- 인벤토리 대조를 통해 누락 검증 필수
- **허용 변환**: 구조 재배치, 문체 변환, 표현 정제
- **금지 변환**: 사실 삭제, 의미 변경, 근거 없는 추가

### 2. 내러티브 우선 (Narrative First)

분석 보고서의 섹션별 나열 구조를 서사적 흐름으로 재구성한다.

- 시간순/인과순/대비 구조 중 최적 내러티브 선택
- 독자의 몰입을 유도하는 Hook → 전개 → 귀결 구조
- 데이터와 사례를 서사적 맥락에 자연스럽게 통합

### 3. 미주 분리 (Endnote Separation)

본문의 가독성을 위해 메타데이터와 방법론을 미주로 분리한다.

- 본문: 핵심 사실, 분석, 서사, 인용
- 미주: 출처, 방법론, 신뢰도, 데이터 상세, 반론/재반박 일부
- `[^N]` 형식 마크다운 각주 사용, 번호는 1부터 연속

### 4. 스타일 가이드 준수

`final-report-style-guide.md`의 모든 규칙을 적용한다.

- 내부 코드(WF1, WF2, pSST 등) 완전 제거
- 영문 약어 전체 표기 + 한국어 번역
- STEEPs 카테고리 코드 서술형 변환
- **참조 경로**: `.claude/skills/env-scanner/references/final-report-style-guide.md`

## 변환 워크플로우

### Phase 1: 입력 분석

1. **보고서 읽기**: 원본 보고서 전체를 읽고 구조를 파악한다
2. **유형 판별**: 보고서 유형을 판별한다
   - **통합 일일 보고서**: 다수의 신호를 포괄하는 종합 분석
   - **주제별 심층 보고서**: 단일 주제에 대한 깊이 있는 분석
   - **단독 워크플로우 보고서**: WF1 또는 WF2 단독 결과물
3. **인벤토리 추출**: 원본의 모든 핵심 요소를 목록화한다
   - 신호 제목 목록 (전체)
   - 핵심 데이터 포인트 (수치, 날짜, 비율)
   - 출처 목록 (전체)
   - 논증 구조 (대전제-소전제-결론)
   - 반론 및 재반박
   - 시나리오
   - 전략적 시사점

### Phase 2: 내러티브 구성

`references/narrative-structure.md`를 참조하여 기사 구조를 설계한다.

1. **기사 제목 생성**: 저널리즘 스타일 제목 + 부제
2. **도입부 작성**: Hook 기법 적용 (800-1,200자)
3. **본문 부(部) 구성**: 신호를 서사적 부(部)로 클러스터링
   - `references/content-mapping-rules.md`의 클러스터링 규칙 적용
   - 각 부(部)에 소제목 부여
   - 부(部) 간 전환 문장 작성
4. **에필로그 작성**: 원형 구조, 열린 질문 (600-1,000자)
5. **미주 구성**: 출처 → 방법론 → 신뢰도 → 데이터 순서

### Phase 3: 스타일 정제

1. **스타일 가이드 적용**
   - `.claude/skills/env-scanner/references/final-report-style-guide.md` 읽기
   - 내부 코드 변환 (WF1→일반 환경스캐닝, pSST→신뢰도 등)
   - 영문 약어 전체 표기 변환
2. **톤 조정**
   - Foreign Affairs/The Atlantic 스타일: 권위적이면서 접근 가능한 산문
   - 불필요한 학술 용어 제거, 핵심 학술 개념은 산문적으로 풀어서 설명
   - 표(table) → 산문 또는 서사적 나열로 변환 (핵심 데이터 표는 유지 가능)
3. **문장 정제**
   - 수동태 최소화
   - 구체적 수치/사례 우선
   - 문단 길이 3-5문장 유지
   - 전문 용어 첫 등장 시 설명 병기

### Phase 4: 무손실 검증 + 출력

1. **인벤토리 대조**
   - Phase 1에서 추출한 인벤토리의 모든 항목이 본문 또는 미주에 존재하는지 확인
   - 누락 항목 발견 시 Phase 2로 돌아가 보완
2. **내부 코드 검증**
   - 최종 파일에서 "WF1", "WF2", "pSST", "SKEL-", "SIG-" 검색 → 0건 확인
3. **미주 무결성 검증**
   - `[^N]` 번호 연속성 확인
   - 본문 참조와 미주 정의 1:1 매칭 확인
4. **파일 저장**
   - 파일명: `{원본파일명에서 유형을 장편저널리즘으로 변경}.md`
   - 저장 경로: 원본과 동일 디렉토리
5. **최종 검토용 열기**
   - `open` 명령으로 파일 열어 사용자 확인 요청

## 품질 체크리스트

### A. 구조 (Structure)

- [ ] 기사 제목이 저널리즘 스타일인가 (보고서 제목과 다른가)
- [ ] 도입부에 Hook이 있는가 (장면/통계/비유 중 하나)
- [ ] 본문이 부(部) 구조로 나뉘어 있는가
- [ ] 에필로그가 도입부와 호응하는가 (원형 구조)
- [ ] 미주가 본문 뒤에 분리되어 있는가

### B. 무손실 (Lossless)

- [ ] 원본의 모든 신호 제목이 본문 또는 미주에 존재하는가
- [ ] 핵심 데이터 포인트(수치, 비율, 날짜)가 보존되었는가
- [ ] 모든 출처가 미주에 기록되었는가
- [ ] 논증 구조(대전제-소전제-결론)가 서사적으로 표현되었는가
- [ ] 반론 및 재반박이 포함되었는가

### C. 스타일 (Style)

- [ ] 내부 코드(WF1, WF2, pSST)가 0건인가
- [ ] 영문 약어에 한국어 번역이 병기되었는가
- [ ] 톤이 Foreign Affairs/The Atlantic 수준인가
- [ ] 수동태가 최소화되었는가
- [ ] 문단 길이가 3-5문장인가

### D. 미주 (Endnotes)

- [ ] `[^N]` 번호가 1부터 연속적인가
- [ ] 본문 참조와 미주 정의가 1:1 매칭되는가
- [ ] 미주 순서: 출처 → 방법론 → 신뢰도 → 데이터인가
- [ ] 미주에 내부 코드가 없는가

### E. 가독성 (Readability)

- [ ] 전문 용어 첫 등장 시 설명이 있는가
- [ ] 부(部) 간 전환이 자연스러운가
- [ ] 도입부가 800-1,200자 범위인가
- [ ] 에필로그가 600-1,000자 범위인가

### F. 최종 검증 (Final)

- [ ] 파일명이 규칙에 맞는가
- [ ] 저장 경로가 원본과 동일한가
- [ ] `open` 명령으로 사용자에게 전달되었는가

## 제한사항

- 원본 보고서의 사실(fact)을 변경하거나 삭제할 수 없다
- 원본에 없는 정보를 추가할 수 없다 (서사적 연결 문장은 허용)
- 내부 워크플로우 코드를 최종 산출물에 남길 수 없다
- 미주 번호를 건너뛸 수 없다 (1, 2, 3, ... 연속)
