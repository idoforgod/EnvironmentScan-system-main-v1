# 콘텐츠 매핑 규칙 (Content Mapping Rules)

> **용도**: 환경 스캐닝 보고서의 각 섹션/필드를 장편 저널리즘 기사의 어디에
> 배치할지 정의하는 규칙.
>
> **참조 스킬**: longform-journalism

---

## 1. 섹션 매핑 테이블

원본 보고서의 섹션을 기사의 어느 위치에 배치할지 정의한다.

### 1.1 통합 일일 보고서

| 원본 섹션 | 기사 위치 | 처리 방식 | 미주 이동 항목 |
|-----------|-----------|-----------|---------------|
| 경영진 요약 | 도입부 | 핵심 발견을 Hook에 통합 | 세부 수치 |
| 분석의 전제와 방법론 | 미주 | 전체 미주로 이동 | 전체 |
| 확실한 결과 N | 제N부 본문 | 서사적으로 재구성 | 신뢰도 점수 |
| 논증 구조 | 제N부 본문 | 대전제-소전제-결론을 산문으로 | 형식적 논증 구조 |
| 근거 A, B, C... | 제N부 본문 | 서사적 증거로 통합 | 상세 출처 |
| 반론 검토 및 재반박 | 제N부 본문 | 본문 흐름에 자연스럽게 통합 | - |
| 소결 | 제N부 마무리 또는 전환부 | 부(部) 결론으로 | - |
| 기존 신호 업데이트 | 본문 또는 미주 | 핵심 변화만 본문, 나머지 미주 | 상세 상태 변화 |
| 패턴 및 연결고리 | 본문 (전용 부 또는 분산) | 서사적 연결로 변환 | 기술적 도표 |
| 전략적 시사점 | 제N부 또는 에필로그 | 서사적 전망으로 | 구체적 액션 리스트 |
| 플러서블 시나리오 | 본문 (미래 전망 부) | 시나리오를 서사적으로 묘사 | 확률 수치 상세 |
| 신뢰도 분석 | 미주 | 전체 미주로 이동 | 전체 |
| 부록 | 미주 | 선택적 통합, 대부분 미주 | 전체 신호 목록, 출처 목록 |

### 1.2 주제별 심층 보고서

| 원본 섹션 | 기사 위치 | 처리 방식 |
|-----------|-----------|-----------|
| 요약/개요 | 도입부 | Hook + 핵심 명제 |
| 배경 분석 | 제1부 | 현상 기술 |
| 심층 분석 | 제2-3부 | 원인/영향 서사 |
| 전망/시나리오 | 제4부 | 미래 전망 서사 |
| 참고문헌 | 미주 | 서지 정보 |

## 2. 신호 9필드 분배 규칙

환경 스캐닝 신호의 표준 9개 필드를 본문과 미주로 분배한다.

### 본문에 포함하는 필드

| 필드 | 본문 처리 방식 |
|------|---------------|
| **신호 제목** | 서사적 문장으로 변환하여 본문에 포함 |
| **핵심 내용** | 본문의 핵심 서사 요소 |
| **전략적 시사점** | 서사적 전망/분석으로 통합 |

### 미주로 이동하는 필드

| 필드 | 미주 처리 방식 |
|------|---------------|
| **출처** | `출처: [기관], "[제목]", [매체], [날짜]` |
| **도메인(STEEPs)** | `분류: [한국어 도메인명]` |
| **중요도** | `평가: [등급] (신뢰도 N점)` |
| **신뢰도 점수** | `신뢰도: N점/100점. [차원별 상세]` |
| **모니터링 지표** | `향후 모니터링: [지표 목록]` |
| **시간적 범위** | `적용 시점: [기간]` |

### 처리 원칙

- **핵심 사실(core fact)**: 반드시 본문
- **맥락/상세 설명**: 본문에 통합 (분량이 과도하면 일부 미주)
- **메타데이터**: 반드시 미주
- **추론/분석**: 본문 (저자의 분석적 서사로)

## 3. 미주 구성 순서

미주는 본문 등장 순서에 따라 번호를 부여하되, 내용은 다음 범주로 분류된다.

### 범주별 순서 (혼재 시 본문 등장 순서 우선)

```
[^1] ~ [^K]: 출처 미주 (인용된 보고서, 논문, 미디어)
[^K+1] ~ [^M]: 방법론 미주 (분석 방법, 데이터 수집)
[^M+1] ~ [^N]: 신뢰도 미주 (신뢰도 점수, 등급)
[^N+1] ~ [^Z]: 데이터 미주 (통계, 표, 부록적 정보)
```

### 미주 작성 형식

#### 출처 미주

```
[^1]: Foreign Affairs, "The Unsettling Implications of Xi's Military Purge", 2026.
```

#### 방법론 미주

```
[^15]: 이 보고서는 이중 워크플로우 스캐닝을 적용했다. 일반 환경스캐닝은
20개 이상의 미디어, 싱크탱크, 정책 소스를 독립적으로 조사했으며, 학술 심층 분석은
학술 논문과 연구 보고서를 별도로 분석한 후 통합했다.
```

#### 신뢰도 미주

```
[^18]: 이 신호의 신뢰도는 92점(100점 만점)으로, 분석 대상 15개 신호 중
가장 높다. 복수의 독립 소스에서 동일 사실이 확인되었으며, 중국 국방부
공식 발표에 기반한다.
```

#### 데이터 미주

```
[^22]: 중앙군사위원회 숙청 현황 상세: 허웨이둥(부주석, 2025.10),
장유샤(부주석, 2026.01), 류전리(합참참모장, 2026.01),
리상푸(국방부장, 2023), 먀오화(정치공작부장, 2024).
```

## 4. 클러스터링 규칙

원본 보고서의 신호들을 서사적 부(部)로 묶는 기준.

### 4.1 클러스터링 우선순위

1. **인과적 연결**: 원인-결과 관계가 있는 신호를 동일 부(部)에 배치
2. **주제적 유사성**: 동일 도메인/주제의 신호를 묶음
3. **논증 구조 호응**: 대전제-근거가 동일한 신호를 묶음
4. **시간적 연속**: 동일 사건의 시간적 전개를 추적

### 4.2 클러스터링 적용 예시

**중국 군부 숙청 보고서 (15개 신호)**:

| 부(部) | 포함 신호 (예시) | 클러스터 기준 |
|--------|-----------------|--------------|
| 제1부 | 중앙군사위원회 와해, 인민해방군 전투력 약화 | 인과적 연결 (숙청→약화) |
| 제2부 | 역량-충성 트레이드오프, 주인-대리인 문제 | 주제적 유사 (구조적 메커니즘) |
| 제3부 | 스탈린 논리, 엘리트 결속 파괴, 두려움의 역설 | 인과적 연결 (신뢰 약화 메커니즘) |
| 제4부 | 방위산업 타격, 핵전력 위기 | 주제적 유사 (경제/기술 영향) |
| 제5부 | 시나리오들, 전략적 시사점 | 시간적 연속 (미래 전망) |

### 4.3 단일 부(部) 최대 신호 수

- 권장: 2-4개 신호
- 최대: 5개 (5개 초과 시 부(部) 분리 필요)

## 5. 표(Table) 처리 규칙

### 5.1 산문 변환 대상

다음 유형의 표는 산문으로 변환한다:

- 신호 요약 표 → 서사적 나열
- 상태 변화 표 → 전후 대비 문장
- 소규모 비교 표 (3행 이하) → 산문적 대비

### 5.2 표 유지 대상

다음 유형의 표는 본문에 유지할 수 있다:

- 역사적 선례 비교 표 (4행 이상, 다차원 비교)
- 숙청 현황 표 (구체적 데이터 밀도가 높은 경우)
- 시나리오 비교 표 (복수 시나리오 대조)

### 5.3 미주 이동 대상

다음 유형의 표는 미주로 이동한다:

- 신뢰도 분석 표
- 워크플로우 실행 통계
- 전체 신호 목록 (부록)

## 6. 무손실 검증 방법

### 6.1 인벤토리 대조 절차

1. **신호 인벤토리**: 원본의 모든 신호 제목을 추출 → 리스트 생성
2. **데이터 인벤토리**: 핵심 수치(%, 점수, 날짜, 금액)를 추출 → 리스트 생성
3. **출처 인벤토리**: 모든 출처명을 추출 → 리스트 생성
4. **대조**: 기사 본문 + 미주에서 각 인벤토리 항목의 존재 여부 확인
5. **누락 처리**: 누락 항목 발견 시 본문 보완 또는 미주 추가

### 6.2 자동 검증 항목

최종 파일에 대해 다음 문자열 검색을 수행한다:

| 검색 대상 | 기대 결과 | 위반 시 조치 |
|-----------|-----------|-------------|
| "WF1" | 0건 | 한국어 서술로 변환 |
| "WF2" | 0건 | 한국어 서술로 변환 |
| "pSST" | 0건 | "신뢰도"로 변환 |
| "SKEL-" | 0건 | 제거 |
| "SIG-" | 0건 | 제거 |
| "Grade A" | 0건 | "A등급"으로 변환 |

### 6.3 미주 무결성 검증

1. 본문의 `[^N]` 참조 목록 추출
2. 미주 섹션의 `[^N]:` 정의 목록 추출
3. 두 목록의 1:1 매칭 확인
4. 번호 연속성 확인 (1, 2, 3, ... 빈 번호 없음)

## 7. 특수 콘텐츠 처리

### 7.1 인용문

원본의 직접 인용은 기사에서도 직접 인용으로 유지한다.

```
보고서: 브루킹스 연구소 전문가는 "당 내에 안전지대는 없다"고 분석했다.
기사: 브루킹스 연구소의 한 전문가는 단호하게 말했다. "당 내에 안전지대는 없다."
```

### 7.2 다이어그램/플로차트

텍스트 기반 다이어그램(코드 블록 형태)은 산문적 설명으로 변환한다.

```
보고서:
  중앙군사위원회 와해
    ├─→ 인민해방군 전투력 약화
    └─→ 당내 신뢰 약화

기사:
  중앙군사위원회의 와해는 두 갈래의 연쇄 반응을 촉발했다.
  하나는 인민해방군의 전투력 약화이고, 다른 하나는 ...
```

### 7.3 시나리오

시나리오는 서사적으로 묘사하되, 확률은 미주 또는 괄호로 처리한다.

```
보고서: 시나리오 A: "공고한 고립" (45%)
기사: 가장 유력한 시나리오는 "공고한 고립"이다.[^N] 시진핑이 ...

[^N]: 전문가 평가에 기반한 시나리오 확률: 공고한 고립 45%, 조용한 재조정 30%,
가속적 불안정 20%, 전략적 전환 5%.
```
