# 내러티브 구조 가이드 (Narrative Structure Guide)

> **용도**: 환경 스캐닝 보고서를 장편 저널리즘으로 변환할 때의 서사적 아키텍처 정의.
>
> **참조 스킬**: longform-journalism

---

## 1. 기사 제목 (Title)

### 형식

```
# [저널리즘 스타일 메인 제목]

**[부제: 원본 보고서의 핵심 명제를 한 문장으로]**
```

### 제목 작성 원칙

- **은유/역설 활용**: "숙청할수록 약해지는 제국", "물로 채운 핵미사일"
- **행위자 + 행위 구조**: 주어와 동사가 있는 능동적 제목
- **단어 수**: 한국어 기준 10-20자 (공백 포함)
- **원본 보고서 제목과 달라야 한다**: 학술/분석 제목 → 저널리즘 제목으로 변환

### 부제 작성 원칙

- 원본의 **핵심 명제(thesis)**를 한 문장으로 압축
- 메인 제목이 은유적이면 부제는 직설적으로
- 메인 제목이 직설적이면 부제는 맥락을 추가

## 2. 도입부 (Opening)

### 분량

800-1,200자 (한국어 기준, 공백 포함)

### Hook 기법 (택 1)

#### 기법 A: 장면 묘사 (Scene Setting)

구체적 시간/장소/인물로 시작하여 독자를 현장에 데려간다.

```
2026년 1월, 베이징의 중난하이에서 한 장군의 이름이 사라졌다.
장유샤(張又俠). 시진핑의 아버지 시중쉰과 함께 전장을 누빈
혁명 원로의 아들이자, 시진핑이 정년을 넘겨서까지 곁에 두었던
유일한 장성이었다. ...
```

#### 기법 B: 충격적 통계 (Startling Statistic)

핵심 데이터로 시작하여 독자의 호기심을 자극한다.

```
71.4%. 2022년 시진핑이 직접 구성한 중앙군사위원회 7인 중
숙청된 비율이다. 문화대혁명 이후 단일 임기 내에 군 최고
지도부가 이 정도로 와해된 전례는 없다. ...
```

#### 기법 C: 역사적 비유 (Historical Analogy)

과거의 유사 사건을 현재와 병치한다.

```
1937년 여름, 스탈린은 붉은 군대의 장교단 3분의 2를 숙청했다.
4년 뒤 독일이 침공했을 때, 소련군은 초전에 참패했다. 90년 후,
베이징에서 놀랍도록 유사한 장면이 펼쳐지고 있다. ...
```

### 도입부 구조

1. **Hook** (2-3문장): 위 기법 중 하나로 독자 포착
2. **맥락 설정** (3-4문장): 사건의 배경과 규모 설명
3. **핵심 명제 제시** (2-3문장): 이 기사가 말하고자 하는 바
4. **로드맵** (1-2문장): 이후 전개 방향 암시 (선택적)

## 3. 본문 부(部) 구조 (Body: Parts)

### 부(部)의 기본 단위

```
## 제N부: [서사적 소제목]

[본문 1,500-3,000자]
```

### 소제목 작성 원칙

- **서사적 표현**: "지휘부가 사라진 군대", "두려움이 충성을 대체할 때"
- **보고서 섹션 제목의 직접 사용 금지**: "근거 A: 지휘구조의 물리적 와해" → 불가
- **독자의 다음 페이지 동기 부여**: 소제목만으로 내용을 예측할 수 있되, 읽고 싶게

### 부(部) 내부 구조

1. **도입 문단**: 이 부(部)의 핵심 주장을 한 문단으로 (3-4문장)
2. **증거 전개**: 데이터, 사례, 인용을 서사적으로 배치 (3-5 문단)
3. **반론 통합**: 해당 주제의 반론을 본문 흐름에 자연스럽게 통합
   - "물론 반론도 있다. [반론]. 그러나 [재반박]." 형식
4. **소결/전환**: 다음 부(部)로의 자연스러운 연결 (1-2문장)

### 부(部) 간 전환 기법

- **인과 전환**: "이러한 군사적 약화는 필연적으로 다른 결과를 낳는다."
- **대비 전환**: "그러나 더 깊은 문제는 군대 밖에 있다."
- **확대 전환**: "이 현상을 더 넓은 맥락에서 보면,"
- **시간 전환**: "이 이야기의 뿌리는 수십 년 전으로 거슬러 올라간다."

## 4. 에필로그 (Epilogue)

### 분량

600-1,000자 (한국어 기준, 공백 포함)

### 원형 구조 (Circular Structure)

도입부의 장면/이미지/인물로 돌아가되, 변화된 관점으로 마무리한다.

```
도입부: "장유샤의 이름이 사라졌다."
에필로그: "장유샤가 사라진 자리에 남은 것은 ..."
```

### 에필로그 구조

1. **도입부 회귀** (1-2문장): 도입부의 이미지/인물/장면 재소환
2. **핵심 종합** (3-4문장): 기사 전체의 핵심 결론을 산문적으로 종합
3. **열린 질문** (2-3문장): 독자가 계속 생각하게 만드는 미해결 질문
4. **마지막 문장**: 여운을 남기는 한 문장

### 열린 질문 예시

- "시진핑은 이 역설을 인식하고 있을까? 아니면, 모든 독재자가 그러하듯, 이미 그 역설의 일부가 되어버린 것일까?"
- "다음 숙청이 있을 때, 남은 장군들은 무엇을 생각할까?"

## 5. 미주 (Endnotes)

### 형식

본문 내 참조: `[^1]`, `[^2]`, ...

문서 하단 미주 섹션:

```
---

## 미주

[^1]: [미주 내용]
[^2]: [미주 내용]
...
```

### 미주 배치 순서

미주는 다음 범주 순서대로 그룹화한다 (본문 등장 순서 우선):

1. **출처 미주**: 인용된 보고서, 논문, 미디어 기사의 서지 정보
2. **방법론 미주**: 분석 방법론, 데이터 수집 절차
3. **신뢰도 미주**: 신호 신뢰도 점수, 등급, 차원별 분석
4. **데이터 미주**: 상세 데이터 표, 통계, 부록적 정보

### 미주 작성 원칙

- 한 미주에 한 정보 단위 (너무 길면 분리)
- 출처 미주: `저자/기관, "제목", 매체, 날짜` 형식
- 방법론 미주: 원본 보고서의 방법론 섹션을 요약하여 기술
- 신뢰도 미주: "이 분석의 신뢰도는 N점(100점 만점)으로 평가되며, ..." 형식
- 데이터 미주: 표 형태 유지 가능 (본문에서 산문으로 풀었을 때)

## 6. 톤 가이드 (Tone Guide)

### 목표 톤

**Foreign Affairs + The Atlantic**: 권위적이면서 접근 가능한 산문

### 구체적 기준

| 항목 | 지향 | 지양 |
|------|------|------|
| 문체 | 서술적 산문 | 불릿 포인트 나열 |
| 어조 | 분석적, 절제된 | 감정적, 선정적 |
| 주장 | 근거 기반, 유보적 | 단정적, 예언적 |
| 인용 | 전문가 직접 인용 통합 | 인용 블록 과다 사용 |
| 수치 | 맥락 내 자연스럽게 | 표의 기계적 나열 |
| 독자 | 교양 있는 비전문가 | 학술 전문가만 |

### 표현 변환 예시

| 보고서 표현 | 저널리즘 표현 |
|-------------|--------------|
| "신뢰도 92점" | "복수의 독립 소스에서 확인된 사실로, 이 분석의 신뢰도는 가장 높다" |
| "정치(Political) 도메인" | "정치적 영역" 또는 맥락에서 자연스럽게 |
| "영향 확신도 82점" | 미주로 이동 |
| "CRITICAL 등급" | "가장 시급하고 중대한" |
| "시나리오 A (45%)" | "가장 가능성 높은 시나리오(전문가 평가 45%)" |

## 7. 보고서 유형별 부(部) 구성 가이드

### 7.1 통합 일일 보고서 → 장편 저널리즘

다수의 신호를 포괄하는 보고서의 경우:

```
도입부: Hook + 핵심 명제
제1부: 가장 충격적/중요한 발견 (상위 1-2개 신호 중심)
제2부: 제1부의 원인/배경 심층 분석 (학술 프레임워크 통합)
제3부: 연쇄적 영향 또는 대비되는 발견 (경제, 안보 등)
제4부: 미래 전망 (시나리오, 전략적 시사점)
에필로그: 원형 구조 + 열린 질문
미주
```

### 7.2 주제별 심층 보고서 → 장편 저널리즘

단일 주제를 깊이 파는 보고서의 경우:

```
도입부: Hook + 핵심 명제
제1부: "무슨 일이 일어나고 있는가" (현상 기술)
제2부: "왜 이런 일이 일어나는가" (원인 분석)
제3부: "이것이 왜 중요한가" (영향과 시사점)
제4부: "앞으로 어떻게 될 것인가" (시나리오와 전망)
에필로그: 원형 구조 + 열린 질문
미주
```

### 7.3 부(部) 수 가이드라인

| 원본 신호 수 | 권장 부(部) 수 | 비고 |
|-------------|---------------|------|
| 5개 이하 | 2-3부 | 핵심에 집중 |
| 6-10개 | 3-4부 | 표준 구성 |
| 11-15개 | 4-5부 | 클러스터링 필수 |
| 16개 이상 | 5-6부 | 과도한 길이 주의 |
