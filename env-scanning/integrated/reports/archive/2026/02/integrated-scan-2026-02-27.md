### 8.3 Methodology

**Phase 1 (Research)**: Each workflow independently scans its designated sources within the shared 24-hour scan window. WF1 uses multi-source web search and direct access. WF2 scans arXiv listings and papers. WF3 crawls Naver News sections. WF4 crawls 43 direct news sites in 11 languages. Each workflow deduplicates against its own historical signal database using 4-stage cascade (URL, String, Semantic, Entity matching).

**Phase 2 (Planning)**: Each workflow independently classifies signals into STEEPs categories (WF3/WF4 additionally use FSSF 8-type, Three Horizons, and Tipping Point frameworks). Cross-impact analysis identifies signal interactions within each workflow. pSST scoring ranks signals by composite priority.

**Phase 3 (Implementation)**: Each workflow generates its independent report using skeleton-fill methodology. Reports are validated against workflow-specific profiles. The integration step (this report) merges four final reports using unified pSST normalization (WF3/WF4 scores multiplied by 10), cross-workflow deduplication, and cross-validation analysis.

**Quality Assurance**: 4-layer defense (L1: skeleton-fill, L2: validate_report.py, L3: progressive retry, L4: golden reference 9-field check). Each signal in this report includes all 9 required fields: Classification, Source, Key Facts, Quantitative Metrics, Impact, Detailed Description, Inference, Stakeholders, Monitoring Indicators.
